# seuNome-aula05-pi01
Atividade da aula 
Criação do repositório

É o primeiro passo para começar um projeto no GitHub.
O repositório é como uma pasta online onde ficam guardados os arquivos do projeto e todo o histórico de alterações.

Clone do repositório

Clonar significa copiar o repositório do GitHub para o meu computador.
Assim eu consigo trabalhar nos arquivos mesmo estando offline.

Criação de branch

Branch é como se fosse uma “cópia” do projeto para fazer alterações sem mexer na versão principal.
Isso ajuda a evitar erros e manter tudo organizado.

Registro de commits

O commit é o registro das alterações que eu fiz no projeto.
É como salvar o progresso com uma pequena descrição do que foi feito.

Envio das alterações (push)

O push serve para enviar as alterações do meu computador para o GitHub.
Assim o repositório online fica atualizado.

Atualização do repositório local (pull)

O pull é usado para atualizar meu projeto no computador com as mudanças que estão no GitHub.

Criação de Issues

As Issues servem para organizar tarefas, anotar problemas ou melhorias no projeto.

Organização com Milestone

Milestone é como um objetivo ou etapa do projeto.
Ela agrupa várias tarefas relacionadas a uma mesma meta.

Gerenciamento com Project

O Project funciona como um quadro de organização, onde dá para separar tarefas em “a fazer”, “fazendo” e “concluído”

Abertura de Pull Request

Pull Request é quando eu peço para juntar as alterações de uma branch com a principal.
Antes disso, as mudanças podem ser revisadas.

Realização de Merge

O merge é quando as alterações são finalmente juntadas na branch principal, concluindo o trabalho.
